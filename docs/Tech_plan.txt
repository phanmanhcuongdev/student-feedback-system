DỰ ÁN: HỆ THỐNG KHẢO SÁT Ý KIẾN SINH VIÊN (STUDENT FEEDBACK SYSTEM)
MỤC TIÊU: PHỤC THÙ MÔN TTCS - KẾT HỢP GIỮA HỌC THUẬT & GIẢI PHÁP THỰC TẾ
================================================================================

1. CORE TECH STACK (ENTERPRISE STANDARD)
--------------------------------------------------------------------------------
- Frontend: React TypeScript, TailwindCSS, Shadcn/UI (Giao diện hiện đại).
- Backend: Spring Boot 3.x, Spring AI (Tích hợp Gemini API).
- Database: 
    + SQL Server: Lưu trữ dữ liệu quan hệ (User, Role, Survey, Questions).
    + Elasticsearch: Index dữ liệu khảo sát phục vụ tìm kiếm & phân tích text.
    + Redis: Caching session và Rate limiting (Bảo mật hệ thống).
- Infrastructure: Docker, Docker Compose (Triển khai trên Workstation Xeon).

2. CÁC TÍNH NĂNG NÂNG CAO (SMART FEATURES)
--------------------------------------------------------------------------------
- AI Chatbot (Gemini): 
    + Hỗ trợ sinh viên giải đáp thắc mắc về quy trình khảo sát.
    + Tóm tắt (Summarize) hàng nghìn ý kiến đóng góp cho quản trị viên.
- Sentiment Analysis:
    + Sử dụng LLM để phân loại ý kiến sinh viên thành: Tích cực / Tiêu cực / Trung lập.
    + Xuất báo cáo trực quan dưới dạng biểu đồ và Word Cloud.
- Real-time Analytics:
    + Cập nhật kết quả khảo sát ngay lập tức (WebSockets).
    + Dashboard giám sát hệ thống (kết hợp ELK Stack nếu có thể).
- Security & Auth:
    + OAuth2 (Google Login - Student Mail).
    + Rate Limiting (Ngăn chặn spam/bot bằng Bucket4j hoặc Redis).

3. CẤU TRÚC HỌC THUẬT (ĐỂ ĂN ĐIỂM TUYỆT ĐỐI)
--------------------------------------------------------------------------------
- Actor Identification (Phải có đủ trong tài liệu):
    + Student: Người thực hiện khảo sát.
    + Staff/Faculty: Người xem kết quả thuộc khoa/ngành.
    + Admin: Quản lý khảo sát, cấu hình hệ thống, quản lý AI Prompt.
    + System: (Actor hệ thống) Tự động xử lý dữ liệu định kỳ.

- Key Diagrams (Cần vẽ kỹ):
    + Use Case Diagram: Phân quyền rõ ràng cho từng Actor.
    + Sequence Diagram: Luồng đi từ React -> Spring Boot -> Gemini -> SQL Server.
    + Class Diagram: Thiết kế Entity chuẩn chỉnh (Tránh lỗi logic database).
    + Deployment Diagram: Mô tả cách chạy Docker trên server cá nhân.

4. CHIẾN THUẬT TRIỂN KHAI (TIMELINE)
--------------------------------------------------------------------------------
- Giai đoạn 1: Thiết kế Database & Vẽ đầy đủ sơ đồ UML (Fix lỗi của môn LTW).
- Giai đoạn 2: Code Backend core (CRUD khảo sát) + Security (JWT/OAuth2).
- Giai đoạn 3: Nhúng Gemini API qua Spring AI & Xử lý Streaming response.
- Giai đoạn 4: Tích hợp Elasticsearch cho tính năng Search & Analytics.
- Giai đoạn 5: Hoàn thiện Frontend & Viết báo cáo (Document is Key).
